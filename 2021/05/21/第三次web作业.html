三、实验过程

1、首先安装git，然后在本地创建learngit文件夹，并cd转到learngit目录下。然后用git init初始化该仓库，为git可管理的仓库。

2、然后在learngit中创建readme.txt文件，每次修改内容，用git add readme.txt将文件交的暂存区stage

3、然后用git commit将暂存区的文件提交到分支。用git status查看文件状态。

4、用git log 查看多次提交的版本记录，用git reset –hard HEAD^回退到上一个版本，用git reset –hard版本号 回到指定版本。

5、用git restore – readme.txt 撤销掉修改。

6、删除文件，则用rm test.txt 删除我添加并提交的文件的，再用git rm test.txt来确认删除，并git commit 提交一个版本。

7、在github上登陆我的账户并创建了一个learngit远程仓库，根据github上网页的提示，在本地用git remote add origin https: //github. com/llz-lailizhen/learngit.git git branch -M main命令，将本地仓库与远程仓库连接起来。再用git push -u origin main 将当前分支推送到远程仓库。

8、在github上创建一个带readme.md文件的仓库gitskils，然后克隆出一个本地库。

四、总结

一、遇到的问题 1、直接用git add git commit等语句会发现提示我 Permission denied(没有权限)，要在这些语句前都加上sudo 第一次还需要输入我的密码。

2、教程上说git checkout – 来撤回修改，实际发现不行，根据终端提示应该是，git restore -- 才能撤回，可能是教程使用的WSL终端和我们版本不一致。

3、教程上说上git remote add origin git@github.com:llz-lailizhen /learngit.git 语句用来关联远程仓库，实际会报错，而github网站上有提示操作，按网站上提示的git remote add origin https://github.com/llz-lailizhen/learngit.git 语句就成功了。后面的克隆语句也是类似的问题，教程报错，按网站提示的正确。

4、教程上说git push -u origin master 来将本地的库推送到远程仓库，但是报错，根据网上搜索GitHub 从2020年 10 月 1 日起，在该平台上创建的所有新的源代码仓库将默认被命名为 “main”，而不是原先的”master”，所以用git push -u origin main才能成功。这些都是教程出现的不适应当前版本。

5、学到了git add一次只能提交一个文件到暂存区，git commit一次会将所有文件暂存区提交到分支中

二、实验心得体会 学会了如果使用Git以及相关的一系列操作，感受到了Git的魅力，Git是一种高效便捷的版本控制系统，熟练掌握Git，对以后的项目开发工作十分重要。如果想熟练运用git命令，一定要结合实际情况多加练习，希望能在以后的使用中多多熟悉。
